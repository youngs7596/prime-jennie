## 1. 배경: 왜 이 분석을 하게 되었나

### 1.1 문제 인식

현재 우리 시스템의 매수 전략(GOLDEN_CROSS, MOMENTUM, DIP_BUY, VOLUME_BREAKOUT, GAP_UP_REBOUND)은 모두 **전통적 기술적 분석의 "가설 → 검증" 패러다임**에 기반합니다:

```
기존 방식:
  "RSI가 30 이하면 반등할 것이다" (가설)
  → 백테스트로 검증
  → 파라미터 조정
  → 실전 적용
```

이 방식의 한계:
- **가설이 사람의 직관에 의존** — 검증 가능한 가설 자체가 제한적
- **파라미터 과적합 위험** — RSI_REBOUND처럼 어떤 파라미터로도 성과가 안 나오는 경우 존재
- **미발견 패턴 존재 가능** — 인간이 떠올리지 못한 피처 조합이 수익을 예측할 수 있음

### 1.2 새로운 접근: 데이터에서 패턴을 먼저 발견

```
새 방식 (본 분석):
  과거 일봉 + 뉴스 감성 데이터에서 "실제 수익이 발생한 시점" 라벨링
  → 해당 시점에서 관측 가능한 모든 피처 계산
  → ML 모델이 어떤 피처 조합이 수익을 예측하는지 탐색
  → 발견된 패턴을 새로운 전략 가설로 활용
```

**핵심 차이**: 가설이 먼저가 아니라, **데이터가 먼저**. ML은 "어떤 조건에서 주가가 올랐는가"를 역추적하여 패턴을 발견하는 **가설 생성기** 역할을 합니다.

### 1.3 GAP_UP_REBOUND 전략과의 관계

직전 세션(rsi-rebound-review-2026-03-11.md)에서 RSI_REBOUND를 폐기하고 GAP_UP_REBOUND를 구현했습니다. GAP_UP_REBOUND는 "갭업 반등"이라는 전통적 기법에 기반한 전략이며, 현재 실전 데이터 수집 중입니다. 이번 분석은 그와 별개로, **데이터 자체에서 우리가 모르는 패턴이 있는지** 탐색하는 것이 목적입니다.

## 3. 분석 방법

### 3.1 라벨링: "좋은 매수 시점"의 정의

각 종목의 각 거래일에 대해 **N일 후 종가 수익률(Forward Return)**을 계산하고, 기준값 이상이면 BUY, 이하이면 AVOID로 라벨링했습니다.

```
Forward Return = (N일 후 종가 - 현재 종가) / 현재 종가 × 100

라벨:
  BUY:     Forward Return ≥ +threshold  (좋은 매수 시점)
  AVOID:   Forward Return ≤ -threshold  (피해야 할 시점)
  NEUTRAL: 그 외                        (유의미한 움직임 없음)
```

여러 윈도우(3/5/10/20일)와 기준값(±2%/±3%/±5%)을 조합하여 총 **6가지 설정**으로 분석했습니다.

### 3.2 피처 엔지니어링: 30개 피처

각 종목의 각 거래일에서 아래 피처를 계산했습니다. 모든 피처는 **해당 시점까지의 과거 데이터만** 사용합니다(미래 정보 미사용).

### 3.3 모델: Random Forest Classifier

```python
RandomForestClassifier(
    n_estimators=300,       # 300개 트리 앙상블
    max_depth=12,           # 트리 깊이 제한 (과적합 방지)
    min_samples_leaf=50,    # 리프 노드 최소 샘플 (노이즈 방지)
    class_weight="balanced", # 클래스 불균형 보정
    random_state=42,
    n_jobs=-1,              # 병렬 처리
)
```

**Random Forest를 선택한 이유:**
- Feature Importance를 자연스럽게 제공 — 어떤 피처가 중요한지 직관적으로 파악
- 과적합에 상대적으로 강건
- 비선형 관계와 피처 간 상호작용 포착 가능
- 주가 예측의 "정확도"보다 **피처 중요도 발견**이 목적이므로 해석 가능한 모델 우선

## 5. 핵심 인사이트 요약

### 인사이트 1: "과열 회피"가 가장 강력한 신호

모든 설정에서 가장 일관되고 강력한 패턴입니다:

```
AVOID(하락) 시점의 공통 프로파일:
  - 직전 20일 수익률: +14~17% (이미 많이 올랐다)
  - 이격률(60일): +15~18% (60일선에서 멀다)
  - MACD 히스토그램: 1,000~1,500 (모멘텀 극도로 과열)
  - RSI: 60~62 (과매수 진입)

→ "최근 많이 올라서 이평선에서 멀리 떨어진 종목에 매수하면 손실 확률 급증"
```

**실행 가능한 적용:**
- 기존 Risk Gate에 **"과열 필터(Overextension Filter)"** 추가
- 이격률(60일) > 15% 또는 직전 20일 수익률 > 15%이면 매수 차단
- 현재 시스템에는 이 조건을 직접 체크하는 Risk Gate가 없음

### 인사이트 2: 기관 순매수가 5일 수익률을 예측

수급 데이터가 3개월(5,682건)밖에 없는데도 **5일 윈도우 멀티윈도우 분석에서 Top 3에 진입**(6.9%)했습니다.

```
기관이 최근 3일간 순매수한 종목 → 5일 후 수익률 양호
```

- 이는 기관의 **정보 우위(information advantage)** 또는 **가격 영향(price impact)**을 반영
- 데이터가 더 쌓이면 검증력이 높아질 것
- 현재 Scout의 supply_demand_score(20pt)에 이미 반영되어 있으나, 별도 Risk Gate나 시그널 조건으로 활용 가능

### 인사이트 3: 뉴스 감성은 장기 보유에 가치

| 보유기간 | 뉴스 감성(7일) 중요도 |
|---------|-------------------|
| 3일 | ~3.5% |
| 5일 | ~3.6% |
| 10일 | ~4.2% |
| **20일** | **~6.3%** |

- 단기(3~5일)에서는 기술적 지표에 압도되지만, **보유기간이 길어질수록 뉴스의 예측력이 급등**
- 뉴스가 반영하는 펀더멘털 변화는 시간이 걸려서 가격에 반영됨
- 현재 Scout의 news_score(10pt)는 적절한 설계. 다만 **Scanner의 실시간 매수 시그널보다 Scout의 종목 선별에 더 적합**

### 인사이트 4: 최적 매수 시점의 프로파일

BUY 라벨 데이터의 중위값(median)을 정리하면:

```
최적 매수 시점 프로파일 (5일 ±3%, Top200):
  RSI(14):        ~55    (과매수도 과매도도 아닌 중립)
  이격률(20일):    ~+2%   (20일선 약간 위)
  이격률(60일):    ~+7%   (60일선 적당히 위)
  MACD 히스토그램:  ~54    (소폭 양수 — 모멘텀 시작 단계)
  볼린저 %B:      ~0.64  (밴드 중앙~약간 위)
  거래량 추세:     ~0.99  (평균 수준)
  20일 변동성:     ~3.0%  (적당한 변동성)
```

> **해석**: "폭등 직전의 과열 상태"가 아니라, **"적당히 상승 추세에 있되 아직 과열되지 않은 상태"**에서 매수하는 것이 5일 후 +3% 이상 수익 확률이 가장 높습니다.

### 인사이트 5: 변동성은 기회이자 위험

ATR 비율과 20일 변동성이 모든 설정에서 1~2위를 차지하는 이유:

- **변동성이 높으면** BUY와 AVOID 모두 많이 발생 (큰 움직임 = 큰 수익 또는 큰 손실)
- 변동성 자체는 방향을 알려주지 않지만, **변동성이 높은 구간에서 다른 피처(이격률, MACD)가 방향을 결정**
- → **"변동성이 높은데 과열되지 않은 종목"이 최적의 매수 후보**

| 기관 순매수 | 시기상조, 1년 후 | 시기상조, 인과관계 의문 | **완전 합치** |
| 뉴스 비중 | 현행 유지 | 현행 유지 | **완전 합치** |
| 후속 우선순위 | D → B | D → B | **완전 합치** |

### 9.4 최종 결정

민지의 방법론적 지적 세 가지가 핵심입니다. 제니는 실용성 관점에서 긍정적 평가를 했지만, 민지가 짚은 통계적 편향 문제는 결과의 신뢰도에 직결되므로 반드시 보완해야 합니다.

#### 즉시 실행 (합의된 사항)

| # | 항목 | 내용 | 근거 |
|---|------|------|------|
| 1 | **Overextension Filter 추가** | Risk Gate에 이격률(60일) 기반 매수 차단 추가 | 제니·민지 모두 찬성 |
| 2 | **국면별 차등 임계치** | 민지 제안 수치 채택 (STRONG_BULL 22% ~ BEAR 10%) | 고정 기준 단일 적용은 양쪽 모두 반대 |
| 3 | **MACD 기준 미포함** | 이격률 단일 기준으로 단순하게 | 민지: 계산 복잡 대비 실익 불명확 |

#### 보류 (데이터 부족)

| # | 항목 | 조건 | 근거 |
|---|------|------|------|
| 4 | 기관 순매수 Scanner 활용 | 6개월~1년 데이터 축적 후 재검증 | 양쪽 합치: 과적합/price impact 구분 불가 |
| 5 | news_score 비중 조정 | 현행 유지 (Scout 10pt) | 양쪽 합치: 현재 설계 적절 |

#### 방법론 보완 (후속 분석 시 반영)

| # | 항목 | 내용 | 근거 |
|---|------|------|------|
| 6 | **라벨링 수정** | 당일 종가 → **익일 시가** 매수 기준으로 변경 | 민지: 실현 불가능한 수익률 편향 |
| 7 | **Permutation Importance 추가** | RF impurity-based 결과를 교차검증 | 민지: 연속형 피처 편향 가능성 |
| 8 | **시계열 안정성 검증** | 2025 H2 학습 → 2026 Q1 검증, 역방향도 실행 | 양쪽 합치: 절대 1순위 후속 과제 |
| 9 | **피처 상호작용 분석** | 복합 조건("변동성+이격률+MACD") 승률 탐색 | 양쪽 합치: 2순위 후속 과제 |

#### 실행하지 않는 것

| 항목 | 이유 |
|------|------|
| 이 모델을 매수 시그널 생성기로 사용 | BUY Precision 44%는 실전 부적합 (민지) |
| 섹터별 분석 | 데이터 부족 (민지) |
| XGBoost 교차검증 | 우선순위 낮음 (양쪽 합치), SHAP과 묶어서 나중에 |

### 9.5 실행 계획

| 단계 | 내용 | 시점 |
|------|------|------|
| **Step 1** | `scripts/mine_signals.py` 방법론 보완 (라벨링 수정 + Permutation Importance) | 다음 세션 |
| **Step 2** | 시계열 안정성 검증 (2025 H2 vs 2026 Q1 교차 검증) | Step 1 직후 |
| **Step 3** | Step 2 통과 시 → Overextension Filter 구현 (국면별 차등 임계치) | Step 2 통과 후 |
| **Step 4** | 백테스트로 필터 효과 검증 (기존 전략 성과 비교) | Step 3 직후 |
| **Step 5** | 검증 통과 시 실전 배포 | 장 종료 후 (15:30 이후) |

> **핵심 원칙**: 민지의 지적대로 방법론 보완(Step 1~2)을 **먼저** 하고, 그 결과가 기존 결론과 일치할 때만 실전 구현(Step 3~5)으로 진행. 방법론에 편향이 있었다면 과열 필터의 기준값도 달라질 수 있으므로, 순서를 바꾸면 안 됨.
