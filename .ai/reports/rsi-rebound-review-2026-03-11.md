## 3. RSI_REBOUND: 비활성화 경위

### 전략 설명

RSI(Relative Strength Index)가 과매도 구간에서 반등하는 시점에 매수하는 역추세(contrarian) 전략입니다.

**트리거 조건:**
```
조건: 직전 RSI ≤ 임계값 AND 현재 RSI > 임계값 (아래→위 교차)

국면별 임계값:
  - BULL/STRONG_BULL: 비활성 (상승장에서 역추세 전략 불필요)
  - SIDEWAYS: 40
  - BEAR: 30
  - STRONG_BEAR: 25
```

## 6. 검토 요청: 반등 포착을 위한 새로운 전략

현재 RSI_REBOUND를 단순 파라미터 조정으로 살리기 어렵다는 결론에 도달했습니다.
하지만 **"폭락 후 반등"은 수익 기회가 매우 큰 패턴**이므로, 이를 포착할 새로운 전략이 필요합니다.

### 6.1 현재 RSI_REBOUND의 구조적 문제

```
현재 로직: "직전 RSI ≤ 임계값 AND 현재 RSI > 임계값"
→ RSI가 특정 값을 아래에서 위로 교차하는 순간 1회만 트리거
```

**문제점:**
- (A) RSI는 **지연 지표** — 14일 기간의 평균을 사용하므로 급락/급등에 느리게 반응
- (B) 1분봉 기반 RSI — 실시간 Scanner에서는 1분봉 데이터로 RSI를 계산하므로 분봉 노이즈에 민감
- (C) **거래량 확인 없음** — 기관/외국인의 반등 매수가 수반되는지 확인하지 않음
- (D) **반등 강도 미확인** — RSI 교차만으로는 진짜 반등인지 데드캣 바운스인지 구분 불가

### 6.2 대안 전략 후보

아래는 "폭락 후 반등"을 포착하기 위한 후보 전략들입니다. 각각의 장단점을 검토해주세요.

#### 후보 A: GAP_UP_REBOUND (갭업 반등)

```
조건:
  1. 전일 종가 대비 당일 시가가 +2% 이상 갭업으로 시작
  2. 전일 등락률이 -3% 이하 (폭락 다음날)
  3. Watchlist 종목 (Scout에서 펀더멘털 검증 완료)
  4. 거래량 비율 >= 1.5배 (반등에 거래량 수반)
  5. 시가 이후 30분간 시가 아래로 내려가지 않음 (Opening Range 유지)
```

**장점:**
- 조건이 명확하고 직관적
- 폭락→갭업 패턴만 포착하므로 노이즈 적음
- 거래량 확인으로 기관 매수 수반 여부 검증

**단점:**
- 갭업 없이 장중 점진적으로 반등하는 패턴 놓침
- 전일 종가 데이터 필요 (현재 Scanner는 이미 보유)

**03-10 적용 시 예상:**
- SK하이닉스: 전일 종가 836,000 → 당일 시가 915,000 (+9.4% 갭업) → 트리거 O
- 삼성전자: 전일 173,500 → 시가 187,600 (+8.1% 갭업) → 트리거 O
- 대한항공: 전일 22,400 → 시가 23,900 (+6.7% 갭업) → 트리거 O

#### 후보 B: OVERSOLD_BOUNCE (과매도 반등 개선형)

```
조건:
  1. RSI(14) < 30 (진정한 과매도 구간, 기존 40에서 대폭 하향)
  2. 당일 또는 직전 2일 내 -5% 이상 급락 존재
  3. 당일 양봉(종가 > 시가) 확인
  4. 거래량 비율 >= 2.0배
  5. Watchlist 종목 + hybrid_score >= 65
```

**장점:**
- RSI 30 미만은 통계적으로 과매도 신뢰도가 높음
- 급락 조건 추가로 "진짜 과매도" 필터링
- 양봉 + 거래량으로 반등 확인

**단점:**
- 급락 당일에는 아직 양봉 미확인 → D+1부터 트리거
- RSI 30 미만이 발생하려면 상당 기간 하락 필요 (단일 폭락으로는 부족할 수 있음)

#### 후보 C: REGIME_SHIFT_BUY (국면 전환 매수)

```
조건:
  1. Council의 국면이 BEAR/STRONG_BEAR에서 SIDEWAYS/BULL로 전환
  2. 전환 당일 또는 D+1
  3. Watchlist 상위 10개 종목
  4. 거래량 비율 >= 1.5배
```

**장점:**
- Council의 LLM 기반 시장 판단 활용 (뉴스, 매크로 데이터 종합)
- 단순 가격 기반이 아닌 펀더멘털 기반 판단
- Council은 하루 2회(07:50, 11:50) 실행되므로 장중 전환도 포착 가능
